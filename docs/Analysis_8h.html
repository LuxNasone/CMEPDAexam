Block that compares not unfolded and unfolded distributions by overlaying the corresponding histograms. Also makes a graphs of normalized measured and simulated distributions. Mainly intended to make stylish graphs and for diagnostic of previous outputs.

Parameters
f1: path to file .root containing not unfolded histograms, assumed to be output of NotUnfolded. Default : "NotUnfolded.root";
f2: path to file .root containing unfolded histograms, assumed to be output of Unfolded. Default : "Unfolded.root";
f3: path to file .root contaning MC histograms, assumed to be output of Response. Default : "Response.root";
outnameName of the output ROOT file where comparison plots are saved. Default : "Comparison.root".
Returns
Void. The function produces comparison plots (TCanvas objects) stored in the output file and accessible via TBrowser.
Note
Requires ROOT framework.
Warning
Input file must be output of respectively CrossSection, Response and Unfolded.

Block that reconstructs the distributions of variables used to express the differential cross-section (Z transverse momentum, rapidity and optimized angle) from data, without any unfolding applied. The function takes input data from a folder containing .root files in NANOAD format, processes the events, and returns three histograms (TH1D) that are used for further analysis.

Parameters
folder_namePath to the input folder cointaining ROOT file with Montecarlo data. File are expected in the NANOAD format.
outnameName for the output file, advised to use a .root file in order to easily see results with a TBrowser. Default : "NotUnfolded.root";
MTBool, if true enables multithreading with the option ROOTEnableImplicitMT(). Default : true;
muteBool that if true disables some secondary comments. Default : false.
Returns
Vector of the three histograms (TH1D) saved on the file called outname, meant to be unfolded later on. In order :
  • [0] : Z tranverse momentum;
  • [1] : Optimzed angle;
  • [2] : Z rapidity absolute value; File are saved and can be visualized with a TBrowser.
Note
Requires ROOT framework.
Warning
Input file must have expected NanoAD format for CMS OpenData.

Block that calculates the response matrices by matching generated and reconstructed events of interest (Z in dimuon). The function builds three RooUnfoldResponse objects (one per observable) and stores them in an output ROOT file, to be used for further analysis.

Parameters
folder_namePath to the input folder cointaining ROOT file with Montecarlo data. Files are expected in the NANOAD format.
outnamePath to the output ROOT file where the response matrices are saved. Default: "Response.root".
MTIf true, enables ROOT implicit multithreading via ROOT::EnableImplicitMT(). Default: true.
muteBool that if true disables some secondary comments. Default: false.
Returns
Void. The function writes RooUnfoldResponse objects and related histograms to the output ROOT file, accessible via TBrowser.
Note
Requires ROOT framework and RooUnfold package.
Warning
Input file must have expected NanoAD format for CMS OpenData.

CrossSection wrapper that applies Bayesian unfolding (RooUnfoldBayes) to three reconstructed histograms, using a precomputed response matrix.

Parameters
folder_name: Path to the input folder cointaining ROOT file with Montecarlo data. Files are expected in the NANOAD format;
n_iterNumber of iterations for the Bayesian unfolding algorithm.
rpathPath to the ROOT file containing the response matrix (generated with Response.cpp). The matrices must be compatible in binning and observable definition. Default : "Response.root"
outnameName of the output ROOT file where unfolded histograms will be stored. Default : "Unfolded.root"
Returns
Void. The function writes Unfolded histograms to the output ROOT file, accessible via TBrowser.
Note
Requires ROOT framework and RooUnfold package.
Warning
Input file must have expected NanoAD format for CMS OpenData.

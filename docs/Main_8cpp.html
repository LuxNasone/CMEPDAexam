A measurement of distribution for variables used to express the differential cross-section (Z transverse momentum, rapidity and optimized angle), without any unfolding procedures applied. This function reads input data from a ROOT file, processes the events, and produces three histograms (TH1D) that can be used for further analysis (e.g. unfolding).

Parameters
fname: file path, required to be a .root file;
outname: name for the output file, required to be a .root file. Default : "Repo/outFiles/NotUnfolded.root";
MT: bool that if true enables multithreading with the option ROOTEnableImplicitMT(). Default : true;
mute: bool that if true disables some secondary comments. Default : false.
Returns
a vector of the three histograms (TH1D) saved on the file called outname, meant to be unfolded later on. In order : [0] : Z tranverse momentum [1] : Optimzed angle = tan(($\pi$ - $\Delta_{\phi}$)/2)/cosh($\Delta_{\eta}$/2) [2] : Z rapidity Advised to visualize with a TBrowser.
Note
Requires ROOT framework.
Warning
Input file must have expected structure, like NanoAD CMS OpenData.

Parameters
fnamePath to the input ROOT file containing reconstructed histograms. The file must contain the expected TTree/histogram structure (see CrossSection).
n_iterNumber of iterations for the Bayesian unfolding algorithm. Typical values range from 2 to 10; higher values may introduce fluctuations.
rpathPath to the ROOT file containing the response matrix (generated with Response.cpp). The matrix must be compatible in binning and observable definition.
outnameName of the output ROOT file where unfolded histograms will be stored.
Returns
std::vector<TH1D*> Vector containing the three unfolded histograms, in the following order: [0] variable X, [1] variable Y, [2] variable Z (see CrossSection for definitions). The caller is responsible for memory management.
Note
Requires ROOT framework and RooUnfold package.
Warning
Input file must have expected structure, like NanoAD CMS OpenData.
